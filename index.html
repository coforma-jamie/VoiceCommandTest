Interactive elements + voice command test page

Note: If you aren't able to see the VADS componenets at the bottom of this page, use this page to test instead.

This page is a playground to test how voice command tools interact with the accessible names of interactive elements, and how they determine those names.

We're testing two sets of components:

  • "Standard" components like a and button
  • VADS components like va-link and va-button

Instructions

  1. Open the audit spreadsheet (opens in new tab)
  2. Pick one of the following tools:
    • who?: Dragon (latest version)
    • who?: Dragon (earlier version)
    • who?: Talon
    • who?: Apple Voice Control (iOS)
    • ✅ Jamie: Apple Voice Control (MacOS)
    • Jamie: Windows Speech Recognition (Windows 10 and earlier)
    • who?: Windows Voice Access (Windows 11)
    • who?: Android Voice Access (unrelated to the Windows feature of the same name)
  3. For each interactive element on the page, using each tool listed above, do the following:
    1. Interact via the visible label (i.e., the visible text within the component)
    2. Interact via the accessible name (accName)
      • elements not using ARIA - the visible label IS the accName
      • for elements using aria-label, the accName is the aria-label
      • elements using aria-labelledby have their accName specified
    3. Interact via a combination of visible label and accName, if applicable
    4. Determine if the test is successful - you are able to use the visual label to activate the element.
    5. Write down your findings in the audit spreadsheet
  4. If you've completed testing all interactive elements using a specific tool, label the tool in the list above with a checkmark: ✅
